Separating web apps from data storage

The web is not as open as it used to be: big monopoly platforms have formed new proprietary layers on top of it. But we create a better architecture for the web. We break the »you get our app, we get your data« package deal with a cross-origin data storage protocol, separating application servers from data storage.

Unhosted is a non-profit project. To know more, check our mailing list and chat room. Our project is now one year old; you can listen to a podcast, or see our tasks list.

remoteStorage: localStorage + sync

  1. Write a localStorage app or take an existing one!
  2. Include remoteStorage.js
  3. Search and replace localStorage with remoteStorage

Done! Now people can use your app with their remote storage provider.

A simple text editor using remoteStorage as backend

<html>
  <head>
    <script src="http://unhosted.org/remoteStorage.js">{category: 'documents'}</script>
  </head>
  <body onload="document.getElementById('textfield').value = remoteStorage.getItem('text');">
    <textarea id="textfield" onkeyup="remoteStorage.setItem('text', this.value);"></textarea>
  </body>
</html>
